# SauceDemo Manual Testing Project

## 📌 Project Overview
This project demonstrates a complete manual testing process for the SauceDemo e-commerce web application. It includes test planning, test case design, defect reporting, and test execution reporting.

---

## 🌐 Application Under Test
https://www.saucedemo.com/

---

## 🎯 Scope of Testing
The following features were tested:

- Login Functionality
- Product Listing
- Product Sorting
- Cart Management
- Checkout Process

---

## 🧪 Types of Testing Performed
- Functional Testing
- UI Testing
- Negative Testing

---

## 👤 Test Users Used
- standard_user
- problem_user
- visual_user

---

## 📂 Test Artifacts Included

This repository contains:

- Test Plan Document
- Test Cases (Markdown + Excel)
- Bug Reports
- Test Execution Report
- Execution Summary

---

## 🐞 Key Defects Identified

- Add to Cart functionality failure for multiple products under error_user
- Minor UI misalignments was observed under visual_user

---

## 🧠 Test Approach

Testing was performed using a structured approach:
1. Requirement analysis
2. Test case design
3. Test execution
4. Defect logging
5. Test summary reporting


---

## 🛠 Tools Used
- Microsoft Excel (Test cases & reports)
- Markdown (Documentation)
- GitHub (Version control & portfolio hosting)

---

## 📊 Summary
The application is functionally stable for core workflows such as login and checkout. However, UI and product sorting issues were identified that require attention.
